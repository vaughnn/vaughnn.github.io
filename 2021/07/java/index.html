Nginx502,Java调试排错

Written by vaughn on 2021-07-31 14:25:32 in Java
| 共1041字，阅读约 3 分钟

前段时间突然nginx大量502,没有日志 top -H -p pid，查看该进程的子线程，可以看到有部分线程CPU占用较高

  1. jstack -l pid > /tmp/jstack_pid.log，看线程栈。线程ID转16进制（在线工具），搜索这几个CPU占用高的线程（线程栈平时更多的是用于排查死锁或其他异常等待的问题，分析线程栈还有一个在线的工具：fastThread.io）。 例如，“810"转16进制为"32a”，发现这是GC的线程，其他的线程"812"， “808"等，经查看，也是GC线程。

  2. 有点Tricky？第三步，看内存。jstat -gcutil pid 1000 5 。上面的线索对应上了，目前总结到的现象是：Eden区和Old区都是100%，GC线程CPU占用高

  3. 初步锁定问题出在内存上。dump堆内存：jmap -dump:format=b,file=/tmp/ServiceName_pid.hprof 。这一步，java进程分配的内存越大，要等待的时间越长，耐心等待吧。把生成好的.hprof文件拷贝下来，好进行后续的分析。

  4. 用 jvisualvm 打开我们的.hprof文件，查看top对象。排名第一的对象是 byte[]，内存占用30%+，这个姑且算正常，不去深究。排名第二的对象是……BigDecimal，内存占用20%+（这个java进程分配了4G内存），这明显是有问题的。由于jvisualvm 太卡，换了个工具，继续死磕我们的 .hprof。

  5. 用 mat（官网）打开我们的.hprof继续分析。 5.1 这会基本已经可以找到问题了

#!/bin/sh
clear
echo "########################################"
echo " 收集java进程诊断信息"
echo "########################################"

echo "请输入jar包名称以获得pid (ps -ef | grep YourInput):"

read jar_name

ps_line=`ps -ef | grep $jar_name | grep -v grep`
pid=`echo $ps_line | awk '{print $2}'`

echo "-----------------------------"
echo "pid: $pid"
echo "ps命令输出(如果未找到唯一行，请退出并重新出入jar包名称)："
echo "$ps_line"
echo "-----------------------------"

echo "是否对当前进程收集诊断信息？(y/n):"

read choice

if [ $choice != 'y' -a $choice != 'Y' ]; then
  echo "退出，请重新执行。"
  exit 0
fi

dir_name="jvm_trace_${jar_name}_`date '+%Y%m%d_%H_%M_%S'`"


echo "创建文件夹：$dir_name"
mkdir $dir_name

echo "打印top信息…………"
top -H -p $pid -b -n 1 > $dir_name/top.log

echo "打印GC信息…………"
jstat -gccause $pid > $dir_name/gc.log
echo "" >> $dir_name/gc.log
echo "----------------------------------------------------" >> $dir_name/gc.log
echo "" >> $dir_name/gc.log
jstat -gc $pid >> $dir_name/gc.log

echo "打印线程栈信息(时间较长，请稍候)…………"
jstack -l $pid > $dir_name/jstack.log

echo "收集堆内存信息(时间很长，请稍候)…………"
jmap -dump:format=b,file=$dir_name/dump_heap.hprof $pid

echo "已完成收集，请查看${dir_name}目录"
